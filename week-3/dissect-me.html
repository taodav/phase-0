It's been an enlightening journey so far. This first week we've been through what's called version control.

What's version control?

Version control is project developer's version of iCloud. Seeing as there's so many developers all working on a single project, there needs to be a system where everyone can back up their work into the cloud, and also retrieve work from the cloud.
Git is the source code management system that we'll be using. Every time you start work on a part of the project, you create a new copy, or 'branch' of the project to work on. After you finish your work for the day and want to include your finished work into the actual project, you merge your branch with the original, or 'master', branch.

So what's so great about version control?

Version control allows multiple developers to simultaneously contribute to the project instead of all working on shared folders. Without version control, if a new version of the project you're working on suddenly stops working because of bugs, then you'll have no 'backup' to go back to. With version control, since your branch is a copy of the original project, the bug will only affect the separate branch that the developer was working on.

So how does Git help you track these changes?

As mentioned above, when using git, all your work will be done on separate branches that are just copies of the original project just for one developer to work on. After you finish work, you will 'commit' the changes you have made to the project by sending in a commit - the 'bundle' of changes you have just sent in to merge with the project. This commit will have a message attached to it - normally this message will have a detailed description on what changes were made and all the details of the change. This is how git tracks all these changes - if you want to revert to a previous instance of your work, then you can see your list of commits and choose which one to reset to.

What is Github and why should I trust it with my code??

Github is an online space for developers to share and code together. Each user has their own profile and their own repositories for their code. It allows you to upload your projects into these repositories so that other people can access this project online. It also allows you to limit who updates your projects by requiring uploaders to make 'pull requests' in order to update the file.
